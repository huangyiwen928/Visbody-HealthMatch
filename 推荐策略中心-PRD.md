# 推荐策略中心 PRD (Product Requirement Document)

> 版本：V1.1  
> 最后更新时间：2026-01-22  
> 状态：编写中

## 1. 模块概览

+ **模块名称**：推荐策略中心 (Recommendation Strategy Center)
+ **核心价值**：提供"人货匹配"的决策大脑配置工具。运营人员可以在此配置推荐规则，系统根据用户画像（元数据）自动匹配最合适的商品（SKU）。
+ **适用角色**：
    - **商品运营**：配置推荐规则、撰写推荐文案。
    - **系统管理员**：管理策略版本、处理线上故障。
+ **上下游关系**：
    - **上游依赖**：
        - **元数据字典**：提供用户标签（检测指标、问卷标签、衍生标签）作为筛选条件。
        - **商品管理中心**：提供产品系列（SPU）和**推荐匹配属性**（SKU属性）。
    - **下游服务**：
        - **H5报告**：展示推荐理由（SAY）和推荐商品列表（DO）。

## 2. 核心概念与逻辑

### 2.1 策略包 (Strategy Package)
**定义**：针对某一特定品类（如枕头、床垫）的完整推荐规则集合。
**规则**：
- **单例模式**：同一品类在同一时间只能有一个生效（Online）的策略包。
- **覆盖机制**：发布新策略时，若该品类已有生效策略，新策略将自动覆盖旧策略，旧策略归档为历史版本。

### 2.2 匹配计算模型（四步漏斗）

当用户完成体测后，系统按以下顺序执行推荐计算：

```mermaid
graph LR
    A[用户画像数据] --> B{1. 熔断过滤}
    B -- 安全通过 --> C{2. 正向匹配}
    B -- 触发禁忌 --> X[剔除特定商品]
    X --> C
    C -- 命中规则 --> D[3. 加权算分]
    D --> E[4. 截断输出]
```

#### 第一步：熔断过滤 (Fuse Breaker) —— 最高优先级
**目的**：确保用户健康安全，绝对禁止推荐禁忌商品。
**逻辑**：`IF 用户命中[禁忌标签] THEN 强制剔除[特定属性]商品`
**示例**：
- 用户标签：`乳胶过敏`
- 剔除动作：排除所有 `材质 = 天然乳胶` 的 SKU。

#### 第二步：正向匹配 (Positive Matching) —— 两级筛选
**目的**：精准匹配适合用户的商品系列和规格。
**执行逻辑**：
1. 按规则优先级（数字越小优先级越高）依次匹配。
2. 每条规则包含两级筛选：
   - **L1 系列锁定 (Series Lock)**：先圈定推荐哪个产品系列（如"深睡护脊枕系列"）。
   - **L2 属性筛选 (Attribute Filter)**：在该系列内，进一步筛选符合条件的 SKU。支持筛选**类目通用属性**（如高度）和**自定义属性**。

**多规则冲突处理**：
- **同一属性冲突**：高优先级规则的属性值覆盖低优先级（如 Rule1定"硬"，Rule2定"软"，取Rule1）。
- **不同属性叠加**：属性条件取并集（如 Rule1定"高度=高"，Rule2定"材质=记忆棉"，结果为"高且记忆棉"）。

#### 第三步：优先级排序 (Priority Sorting)
**V1 简化逻辑**：
不进行复杂的加权算分，采用**“瀑布流”**机制，确保运营配置的规则顺序即为最终推荐顺序。

**执行逻辑**：
1.  系统按照**规则列表的顺序**（Rule 1 > Rule 2 > Rule 3...）依次运行。
2.  **Rule 1 命中**：该规则产生的商品/参数，排在结果列表的**最前方**。
3.  **Rule 2 命中**：其结果排在 Rule 1 的结果**之后**。
4.  以此类推，直到达到 Top N 数量限制。
5.  **规则内排序**：若单条规则输出了多个商品，默认按商品上架时间倒序排列（或在规则内支持简单手动排序）。

#### 第四步：截断输出 (Top N)
- 截取前 N 个结果输出。
- 若结果不足 N 个，触发兜底策略补齐。

### 2.3 兜底策略 (Fallback Strategy)
**定义**：当所有匹配规则都未命中，或命中后无可用商品时的默认行为。
**V1 简化**：仅保留**指定单一 SKU** 作为兜底，不支持榜单与多选。
**可选方案**：
1.  **指定 SKU**：在当前策略包中选择 1 个固定 SKU。

## 3. 功能详细设计

### 3.1 策略列表页
- **展示**：策略名称、关联品类、当前状态（Draft/Online/History）、最后修改时间。
- **操作**：
    - **新建**：打开新建向导。
    - **编辑**：Draft 状态可直接编辑；Online 状态需创建副本编辑。
    - **删除**：仅 Draft 和 History 状态可删除。

### 3.2 新建策略向导
分两步完成策略初始化：
1. **基础信息**：填写策略名称、描述。
2. **选择品类**：**核心约束**。一旦创建，品类不可修改（因为规则依赖特定品类的属性模板）。

### 3.3 策略详情配置页

#### A. 全局设置
- **Top N**：设置最终透出的商品数量（1-10）。
- **兜底策略**：指定单一 SKU 作为兜底。
- **排序偏好**：
    - **匹配度优先**（默认）：严格按照规则匹配程度排序。
    - **热度优先**：在命中规则的商品中，优先展示销量高的。

#### B. 熔断规则配置 (红色区域)
- **触发条件**：多选用户标签（OR 关系）。
- **排除对象**：选择要排除的**推荐匹配属性**（如 材质=乳胶）。

#### C. 匹配规则配置 (蓝色区域)
支持拖拽排序调整优先级。每条规则包含：
- **IF (用户特征)**：
    - 逻辑关系：满足所有(AND) / 满足任一(OR)。
    - 标签选择：引用元数据字典的标签。
- **THEN (推荐方案)**：
    - **输出类型**：选择 **具体商品** 或 **通用参数**。
    - **配置内容**：
        - **商品模式**：
            - 选择系列、筛选属性。
            - **推荐数量**：可选择“跟随全局 Top N”或“指定固定数量”（如仅推1个）。
        - **参数模式**：
            - **参数名**：必须从该品类的**推荐匹配属性模板**中选择（数据闭环），不可手填。
            - **建议值**：可选择属性值或手填区间。
- **SAY (推荐理由)**：
    - **富文本编辑**：支持加粗、颜色、列表等基础排版。
    - **图片插入**：支持插入科普图片或图表，丰富展示效果。
    - **话术模板**：支持引用预设话术。

### 3.4 验证与模拟中心 (Validation Center)
提供三种层级的测试能力，确保策略准确性：

#### 1. 单报告验证 (Single Report Test)
- **场景**：针对特定客诉或典型用户进行排查。
- **输入**：从“体测报告列表”中选择一份真实的历史报告。
- **执行**：读取该报告的用户画像数据，运行当前策略。
- **输出**：展示该用户的推荐结果及匹配路径日志。

#### 2. 批量回测 (Batch Backtest)
- **场景**：发布新策略前，验证对历史存量用户的影响。
- **输入**：选择时间范围（如近30天）或门店范围，提取批量历史报告（如1000份）。
- **执行**：后台批量运行策略。
- **输出**：
    - **结果分布图**：展示各商品/参数的推荐占比（如 A商品 30%，B商品 20%，兜底 10%）。
    - **覆盖率**：成功推荐用户的比例。

#### 3. 预设用例验证 (Preset Case Test)
- **场景**：验证策略逻辑的理论分布，排除脏数据干扰。
- **输入**：使用系统预设的“标准测试集”（包含覆盖各种边缘情况的虚拟用户数据）。
- **输出**：对比“预期分布”与“实际分布”，快速发现逻辑漏洞。

### 3.5 发布校验机制
点击“发布上线”时，系统执行严格校验：
- **阻断性错误**：
    - 未配置任何匹配规则。
    - 规则缺少推荐方案或文案。
    - 引用的标签或商品属性已失效（被删除）。
- **警告性提示**：
    - 未配置熔断规则。
    - 未配置兜底策略。

## 4. 数据模型设计

### 4.1 策略表 (Strategy)
| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 主键 |
| name | String | 策略名称 |
| category_id | String | 关联品类ID |
| status | Enum | draft, online, history |
| top_n | Int | 推荐数量 |
| fallback_config | JSON | 兜底策略配置 |
| version | Int | 版本号 |

### 4.2 规则表 (Rule)
| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 主键 |
| strategy_id | UUID | 外键 |
| type | Enum | fuse(熔断), match(匹配) |
| priority | Int | 优先级（数值越小越高） |
| conditions | JSON | 用户标签条件集合 |
| actions | JSON | 推荐/剔除动作集合 |
| recommend_text | String | 推荐文案 |
| strength | Enum | high, medium, low (推荐力度) |
| logic_type | Enum | and, or |

---

## 5. 接口定义 (API)

- `GET /api/strategies`: 获取策略列表
- `POST /api/strategies`: 新建策略
- `GET /api/strategies/{id}`: 获取详情
- `PUT /api/strategies/{id}`: 更新配置
- `POST /api/strategies/{id}/publish`: 发布（含校验）
- `POST /api/strategies/{id}/simulate`: 运行模拟测试
