# 商品管理中心 - PRD

## 1. 模块概览
+ **模块名称**：商品管理中心 (Product Management)
+ **核心价值**：管理可供推荐策略调用的商品池，将物理商品结构化为可匹配的属性集合，并通过状态控制商品在推荐端的可见性。
+ **适用角色**：商品运营
+ **业务目标**：
    1. 建立可供推荐策略调用的商品池。
    2. 将物理商品结构化为“推荐匹配属性”集合。
    3. 通过手动状态管理，控制商品在推荐端的可见性。

---

## 2. 核心逻辑与定义

### 2.1 信息架构：产品系列 - 单品

采用**双层结构**管理商品，以适应人体工学行业的特性（如同款枕头有不同高度规格）。

#### 产品系列（SPU）
**是什么**：相当于"产品家族"，管理一组相关商品的通用信息。

**包含内容**：
- 系列名称（如"深睡记忆棉枕系列"）
- 所属类目（枕头/床垫/工学椅）
- 系列主图（用于在单品无图时兜底显示）
- 系列简介

**核心规则**：
- 系列决定了"类目"，类目决定了该系列下单品需要填写哪些"推荐匹配属性"
- 一旦系列下创建了第一个单品，**类目即刻锁定不可修改**（防止已录入的属性错乱）

#### 单品（SKU）
**是什么**：具体规格的商品，是推荐策略最终输出的实体。

**包含内容**：
- 基础信息：名称、编码、价格、商品图
- 推荐匹配属性：高度、材质、软硬度等（由类目决定需要填写哪些）
- 售卖状态：控制是否可被推荐策略使用

### 2.2 属性管理体系（核心优化）

为保证数据定义的清晰性和操作的简便性，采用 **"模板定义 + 实例填充"** 的模式。

#### A. 属性分类 (Attribute Types)
在定义类目属性时，必须明确属性的归属类型，以解决"属性定义不清晰"的问题：

1.  **基础规格属性 (Basic Specs)**：
    -   **定义**：描述商品的物理规格，**不参与**推荐算法匹配。
    -   **用途**：在商品详情页展示，供用户查看。
    -   **示例**：包装尺寸、产地、保修期。
    -   **展示位置**：SKU 编辑器 -> 基础信息区。

2.  **推荐匹配属性 (Algo Props)**：
    -   **定义**：描述商品的核心特征，**直接参与**推荐算法的匹配计算。
    -   **用途**：系统根据用户的问卷结果，匹配具有相应属性的商品。
    -   **示例**：高度、软硬度、核心材质。
    -   **展示位置**：SKU 编辑器 -> 推荐匹配属性区。

#### B. 类目属性模板 (Category Template)
在"类目管理"中统一配置。每个类目拥有两份属性清单：
-   **基础规格清单**：如 [包装长, 包装宽, 包装高]
-   **匹配属性清单**：如 [高度, 材质, 软硬度]

#### C. SKU 自定义属性 (Local Custom)
仅在当前 SKU 有效的补充属性。**V1版本不支持自动反哺到类目模板**，以防止门店误操作导致全局数据污染。若需添加全局属性，请前往"类目管理"统一配置。

**操作方式**：
- 点击 "+ 添加自定义属性"。
- 输入 **属性名称**（如：包装尺寸）和 **属性内容**（如：30*40cm）。
- 支持多组属性。

### 2.3 极简状态机 (State Machine)
完全由人工手动控制，无系统自动触发逻辑。

#### 状态定义

| 状态 | 含义 | 推荐策略可见性 | 允许的操作 |
|------|------|----------------|------------|
| **草稿 (Draft)** | 信息不完整或正在编辑中 | 🚫 不可见 | 编辑、删除、复制 |
| **待上架 (Ready)** | 必填属性已填写完整，校验通过 | 🚫 不可见 | 上架、编辑、删除、复制 |
| **已上架 (Online)** | 正式投入推荐池，可被策略匹配 | ✅ **可见** | 下架、编辑(受限)、复制 |
| **已下架 (Offline)** | 从推荐池移除 | 🚫 不可见 | 上架、编辑、删除、复制 |

#### 状态转换条件

| 操作 | 前置条件 | 不满足时提示 |
|------|----------|--------------|
| 保存 → 待上架 | 所有必填属性已填写 | "以下必填属性未填写：[属性列表]" |
| 上架 | 状态为"待上架"或"已下架" | "请先完善商品信息" |
| 删除 | 状态为"草稿"或"已下架" | "已上架商品不可删除，请先下架" |

### 2.4 全局操作规则
1.  **删除规则 (Delete Rules)**：
    -   **删除 SKU**：仅状态为 `草稿` 或 `已下架` 的 SKU 可被删除。**[已上架] 商品严禁删除**，必须先下架。
    -   **删除系列**：仅当该系列下 **没有任何 SKU** 时，才允许删除系列。
    -   **删除类目**：仅当该类目下 **没有任何产品系列** 时，才允许删除类目。否则提示"该类目下存在关联产品，禁止删除"。

2.  **类目锁定规则**：
    -   系列创建时必须指定类目。
    -   一旦系列下创建了第一个 SKU，该系列的 **“类目”字段即刻锁定**。

3.  **属性模板变更规则 (Schema Evolution)**：
    -   **场景**：当类目模板**新增**了一个"必填属性"（如：新增"适用人群"且必填），而该类目下已有 [已上架] 的 SKU。
    -   **规则 - 软兼容**：
        -   **对已上架商品**：**不**会自动下架，状态保持 [已上架]。但前端会标记"属性待完善"。
        -   **触发补全**：当运营人员**下次编辑**该 SKU 时，必须补全该新增字段，否则**禁止保存**。
    -   **目的**：避免因配置调整导致线上商品大面积意外下架。

## 3. 页面功能详细定义

### 3.1 左栏：产品系列管理
+ **列表展示**：
    - **展示元素**：系列主图 (缩略图)、系列名称、包含 SKU 数量。
    - **交互**：点击系列，右侧刷新对应的 SKU 列表。
+ **类目管理与全局属性配置**：
    - **入口优化**：在“现有类目”列表中，每个类目右侧直接提供 **[属性]** 按钮，作为常驻入口。
    - **功能 1：类目维护**：新增/修改/删除类目（如枕头、床垫）。
    - **功能 2：全局属性模板配置**（核心）：
        - 点击 [属性] 按钮后，直接展开该类目的属性配置面板。
        - **配置项详情**：
            1.  **属性名称** (必填)：如 "高度"。
            2.  **属性归属** (必填)：
                *   **基础规格**：仅展示，不参与算法（如：包装尺寸）。
                *   **推荐匹配**：参与算法匹配（如：材质、软硬度）。
            3.  **数据类型** (必填)：
                *   **文本**：自由输入。
                *   **数值**：支持区间（如 10-15）。
                *   **枚举**：单选/多选（需配置选项值）。
            4.  **单位** (选填)：如 "cm"、"kg"，显示在输入框尾部。
            5.  **是否必填**：默认为否。
        - **作用**：明确属性用途，系统根据"属性归属"将字段渲染在 SKU 编辑器的对应区域。

+ **新建系列**：
    - **系列名称** (必填)：限 30 字。
    - **所属类目** (必填)：**核心字段**。下拉选择（枕头/床垫/工学椅）。若该系列下已有 SKU，此项置灰不可选。
    - **默认主图**：必填。建议比例 1:1，JPG/PNG，<2MB。**作用：SKU无图时的默认兜底。**
    - **系列简介** (选填)：富文本，用于前端聚合页展示。
+ **删除系列**：仅当系列下无 SKU 时才允许删除。

### 3.2 右栏：SKU 管理与列表
+ **列表字段**：
    - **复选框：**用于批量操作。
    - **基础信息**：图、名称、SKU编码。注意：若 SKU 未上传图片，显示所属系列的主图。
    - **推荐匹配属性**：展示关键匹配参数（如高度、软硬度）。
    - **状态**：徽标展示（已上架/已下架/待发布/草稿）。
+ **操作**：
    - **单条操作**：编辑、上架/下架、删除。

#### 批量导入 (Batch Import) - V1 核心功能
为解决客户初次初始化数据的痛点，提供**基于类目的模板导入**功能。

+   **入口**：列表页顶部 [导入] 按钮。
+   **流程**：
    1.  **选择类目**：用户选择要导入的目标类目（如：枕头）。
    2.  **下载模板**：系统根据该类目的**属性配置**，动态生成 Excel 模板。
        *   *表头包含*：系列名称(必填)、SKU名称(必填)、SKU编码(必填)、价格、**[该类目的所有推荐属性]**。
    3.  **上传数据**：上传填写好的 Excel 文件。
    4.  **系统处理**：
        *   **系列匹配**：根据 Excel 中的"系列名称"查找已有系列；若不存在，则**自动创建新系列**（使用默认占位图）。
        *   **SKU 创建**：创建 SKU 数据，并将属性值写入。
    5.  **结果反馈**：
        *   导入的数据默认状态为 **[草稿]**，无论属性是否完整（允许先导入后完善）。
        *   若 `SKU编码` 冲突，则**跳过**该条数据并记录失败原因。

#### 批量操作 (Batch Actions)
+ **触发**：勾选列表项后，底部浮出操作栏。
+ **逻辑**：
    - **批量上架**：跳过 [草稿] 和 [无货] 商品，仅处理 [待发布] 和 [已下架]。
    - **批量删除**：**强校验**。若选中的商品包含 [已上架] 状态，系统弹窗报错，**拒绝执行整个批量任务**，提示“包含已上架商品，请先下架”。

#### 批量操作反馈 (Batch Feedback)
+ **逻辑**：前端预判 + 后端执行。
+ **反馈弹窗**：当批量操作包含“不合法对象”时，弹出模态框：
    - 标题：操作完成
    - 内容：成功 N 条，**跳过 M 条**。
    - 详情：列出跳过的 SKU 名称及原因（例：SKU_A - 属性缺失；SKU_B - 无货）。

**批量上架反馈示例**：
```
✅ 批量上架完成
- 成功上架：3 条
- 已跳过：2 条
  - 深睡枕-低款：属性未完善（草稿状态）
  - 护脊枕-软款：属性未完善（草稿状态）
```

**批量删除拦截提示**：
```
⚠️ 无法删除
选中的商品中包含已上架商品，请先下架后再删除。

已上架商品：
- 深睡枕-慢回弹-高款
- 护脊枕-标准版
```

#### 复制 SKU (Duplicate)
+ **入口**：列表操作列 [复制] 按钮。
+ **逻辑**：
    1. 克隆源 SKU 的所有基础信息和推荐匹配属性。
    2. **清空**`SKU 编码`（需重新生成）。
    3. **重置**`SKU 名称` 为 “原名称_副本”。
    4. **重置** 状态为 `草稿`。
    5. 自动打开编辑抽屉。

### 3.3 SKU 编辑器 (抽屉式)
+ **基础信息区**：
    - **固定字段**：名称、编码、价格、商品图。
    - **类目模板字段**：根据类目配置的 **[基础规格属性]** 动态渲染（如：包装尺寸）。
    - **自定义扩展**：支持添加当前 SKU 特有的补充信息。**注意：自定义字段仅当前 SKU 有效，不会同步到类目模板。**
    - **自定义扩展**：支持添加当前 SKU 特有的补充信息。**注意：自定义字段仅当前 SKU 有效，不会同步到类目模板。**

+ **推荐匹配属性区 (按类目动态展示)**：
    - **类目模板字段**：根据类目配置的 **[推荐匹配属性]** 动态渲染（如：高度、材质）。这是推荐算法的核心数据源。
    - **自定义扩展**：支持添加临时匹配属性。

+ **通用操作**：
    - **数值型**：支持精确值或区间值（例：`10-12`）。
    - **枚举型**：下拉选择（例：`记忆棉`）。
    - **校验**：此区域字段未填全，**禁止上架**，只能存为草稿。

## 4. 数据模型与字段规范

### 4.1 产品系列表 (ProductSeries)

| 字段名 | 类型 | 必填 | 说明 | 校验规则 |
|--------|------|------|------|----------|
| id | String(UUID) | ✅ | 系列唯一标识 | 系统自动生成 |
| name | String | ✅ | 系列名称 | 2-30字符 |
| category_id | String | ✅ | 所属类目ID | 关联类目表 |
| cover_image | String | ✅ | 系列主图URL | 图片格式校验 |
| description | Text | - | 系列简介 | 限500字 |
| sku_count | Int | - | 包含SKU数量 | 系统自动计算 |
| created_at | DateTime | ✅ | 创建时间 | 系统自动生成 |
| updated_at | DateTime | ✅ | 最后修改时间 | 系统自动更新 |

**业务约束**：
- 同一账号下系列名称不可重复
- category_id 在系列下有SKU后锁定不可修改

### 4.2 单品表 (SKU)

| 字段名 | 类型 | 必填 | 说明 | 校验规则 |
|--------|------|------|------|----------|
| id | String(UUID) | ✅ | SKU唯一标识 | 系统自动生成 |
| series_id | String | ✅ | 所属系列ID | 外键关联 |
| name | String | ✅ | 商品名称 | 2-50字符 |
| sku_code | String | ✅ | SKU编码 | 2-30字符，全局唯一 |
| price | Decimal | - | 售价（分） | ≥0 |
| image | String | - | 商品图URL | 无图时取系列主图 |
| status | Enum | ✅ | 状态 | draft/ready/online/offline |
| extra_description | Text | - | 扩展描述 | 限500字 |
| created_at | DateTime | ✅ | 创建时间 | 系统自动生成 |
| updated_at | DateTime | ✅ | 最后修改时间 | 系统自动更新 |

### 4.3 SKU属性值表 (SKUAttribute)

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String(UUID) | ✅ | 记录唯一标识 |
| sku_id | String | ✅ | 关联SKU |
| attribute_id | String | - | 关联类目属性定义（如果是模板属性） |
| custom_key | String | - | 自定义属性名（如果是自定义属性） |
| value | String | ✅ | 属性值（数值型存数字，枚举型存选项ID） |
| value_display | String | - | 属性值展示文本 |

### 4.4 类目表 (Category)

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String(UUID) | ✅ | 类目唯一标识 |
| name | String | ✅ | 类目名称 |
| attribute_template | JSON | ✅ | 属性模板配置（包含必填/选填属性列表） |

## 5. 校验规则清单

### 5.1 保存时校验（宽松）

| 检查项 | 规则 | 不通过提示 |
|--------|------|------------|
| 商品名称 | 非空，2-50字符 | "商品名称不能为空" |
| SKU编码 | 非空，2-30字符，全局唯一 | "SKU编码不能为空"或"该编码已存在" |

### 5.2 上架时校验（严格）

| 检查项 | 规则 | 不通过提示 |
|--------|------|------------|
| 必填属性完整性 | 类目要求的所有必填属性都已填写 | "以下必填属性未填写：[属性列表]" |
| 属性值有效性 | 数值在合理范围内 | "属性[属性名]的值不在有效范围内" |

## 6. 验收要点清单

### 功能验收
- [ ] 新建系列时必须选择类目，类目在有SKU后锁定
- [ ] 新建/编辑SKU时，属性表单根据类目动态渲染（继承属性）
- [ ] 支持为SKU添加自定义属性，且保存正常
- [ ] 必填属性未填写时，只能存为草稿，不能上架
- [ ] 已上架商品删除按钮不显示或点击后提示需先下架
- [ ] 复制SKU后，编码清空、名称添加"_副本"、状态重置为草稿
- [ ] 批量删除选中包含已上架商品时，整体拦截不执行

### 状态流转验收
- [ ] 草稿 → 填写完整后可保存为"待上架"
- [ ] 待上架/已下架 → 可上架
- [ ] 已上架 → 可下架
- [ ] 草稿/已下架 → 可删除
- [ ] 已上架 → 不可删除

### 数据一致性验收
- [ ] 删除系列前检查是否有SKU
- [ ] SKU编码全局唯一性校验
- [ ] 类目锁定逻辑生效
